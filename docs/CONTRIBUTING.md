# Contributing

Thank you for considering a contribution to The English Codex.

This is a free public study resource. It gets better when learners, teachers, and developers help improve it.

## Ways to contribute

You can help by:

- reporting grammar mistakes;
- fixing unclear explanations;
- suggesting better examples;
- adding common learner errors;
- improving Copy by Hand blocks;
- improving accessibility;
- fixing HTML, CSS, or JavaScript;
- improving documentation.

You do not need to be a developer to contribute.

## Before contributing

Read the relevant document first:

- content changes: [CONTENT-GUIDE.md](CONTENT-GUIDE.md);
- code changes: [TECHNICAL.md](TECHNICAL.md);
- project direction: [VISION.md](VISION.md);
- reading and accessibility: [READING-EXPERIENCE.md](READING-EXPERIENCE.md).

## Reporting an issue

When opening an issue, include:

- what the problem is;
- where it appears;
- what you expected;
- what happened instead;
- for content corrections, the corrected version and a short reason.

Clear issues are useful. Vague issues are archaeology.

## Making a change

1. Fork the repository.
2. Create a branch with a clear name.
3. Make one focused change.
4. Follow the relevant guide.
5. Test locally by opening the site.
6. Commit using Conventional Commits.
7. Open a pull request explaining what changed and why.

## Commit messages

Use Conventional Commits:

| Prefix | Use for |
|---|---|
| `docs:` | Documentation changes |
| `feat:` | New features |
| `fix:` | Bug fixes or corrections |
| `content:` | Study content additions or updates |
| `style:` | Visual changes that do not affect behavior |
| `refactor:` | Code restructuring without behavior changes |
| `chore:` | Maintenance tasks |

Examples:

```text
content: add A1 page on the verb to be
fix: correct age example in A1 grammar
docs: clarify the notebook review cycle
feat: add topic search
```

Use the imperative mood: `add`, `fix`, `update`, `clarify`.

## Contribution principles

- Write original explanations and examples.
- Do not copy textbooks, courses, websites, official exam tasks, or protected materials.
- Do not add copyrighted images, logos, brand assets, or long quotations.
- Keep pop-culture references as light flavor only.
- Keep the project static, simple, and learner-focused.
- Avoid adding accounts, databases, ads, analytics, or unnecessary dependencies.
